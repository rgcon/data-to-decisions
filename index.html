Math connection

Exponents, logarithms, and derivatives work as one system

Exponents, logarithms, and derivatives are all ways to describe how one quantity depends on another. Exponents model multiplicative dependence, logarithms invert that dependence, and derivatives measure its local rate of change. A clean way to see the connection is to follow one example all the way through.

Suppose a quantity starts at 100 and doubles every hour, and let f(t) denote its value after t hours. Then the three ideas line up naturally: the exponential gives the rule, the logarithm recovers the input from the output, and the derivative tells us how fast the output is changing with respect to time.

Notation in this section is fixed: log2 means a base-2 logarithm because the example doubles, while log without a subscript means the natural logarithm in the derivative formulas.

Exponents

Exponents model multiplicative growth

If the quantity doubles each hour, then after t hours it is f(t) = 100 · 2t. The exponent records how many times the doubling step has happened. Equal increases in time create equal multiplicative changes in the output.

f(t+1) = 2f(t)

One more hour means one more factor of 2. That is the role of the exponent: it counts repeated multiplication.

Logarithms

Logarithms analyze and compare growth

A logarithm reverses the exponential rule. If you want to know when the quantity reaches 800, start with f(t) = 800. Since f(t) = 100 · 2t, substitute that formula to get 100 · 2t = 800. Dividing by 100 gives 2t = 8, so t = log2(8) = 3. The logarithm tells you which exponent produced the observed value.

t = log2(f(t)/100)

This reverse step is why logarithms are useful in growth problems. Here we use log2 because the growth factor is 2, so the logarithm answers "how many doublings happened?"

Derivatives

Derivatives measure rate of change

A derivative measures growth or decay of one quantity with respect to another quantity. Here it tells us how the quantity changes with respect to time, not just over a full hour but at an instant. For this function, f'(t) = 100 · log(2) · 2t = log(2)f(t). So the rate of change is proportional to the current amount: the bigger the quantity becomes, the faster it grows.

f'(t) = log(2)f(t)

In analytic terms this is a rate of change. In geometric terms, if you draw the graph, the same derivative appears as the slope of the tangent line at that point. When the base is e, the formula becomes even cleaner because d/dx ex = ex. The symbol log is used here to denote the natural logarithm, which is the logarithm that appears in derivatives of exponentials.

For example, at t = 1 we have f(1) = 200, so f'(1) = 200 log(2) ≈ 138.6 units per hour. That is the instantaneous rate at t = 1, not the total gain over the next full hour. The average change from t = 1 to t = 2 is (400 - 200)/(2 - 1) = 200 units per hour.

In this example, exponents gave the growth law f(t) = 100 · 2t, logarithms answered the inverse question "when do we reach a target?", and derivatives measured the instantaneous growth rate.

Probability and statistics use the same pattern. Exponentials appear in growth and decay models, logarithms simplify multiplicative relationships such as likelihoods, and derivatives are used to measure sensitivity, rates, slopes, and best-fitting parameters.

multiplicative dependence -> inverse log view -> derivative for local change or optimization

1. Distribution families

Discrete, continuous, and mixed variables

What is a variable?

In statistics and data science, a variable is any quantity that can take different values across observations. When you collect data — number of purchases per customer, body temperature, time spent on a page — each column in your dataset is a variable. Before you model anything you must decide what kind of values it takes, because that determines which probability distribution makes sense, which loss function to optimize, and how to interpret model outputs.

Discrete — countable values Continuous — any value in a range Mixed — spike at a point + continuous tail

Discrete

Countable outcomes

A discrete variable takes values from a finite or countably infinite set — whole numbers, categories, counts. Probability is assigned to each individual outcome as a probability mass.

Probability rule P(X = k) is defined for every value k. All masses sum to exactly 1.
Example: support calls per hour (Poisson λ=3)

Each integer count has its own bar of height P(X = k). Fractions like 2.5 calls are impossible — the variable lives on whole numbers only.

Distribution shape impact

Increasing the Poisson rate λ shifts the mass to the right and spreads it wider. A low λ produces a steep spike near 0. A high λ produces a broad, approximately bell-shaped mass function. You can never ask "what is the probability of exactly 2.7?" — only integer values carry mass.

  • Mean = λ, Variance = λ (Poisson)
  • Chart shows bars at integer values only
  • Real uses: counts of events, defects, arrivals

Continuous

Interval-based probability

A continuous variable can take any value in an interval. No single point carries probability mass — instead probability is spread through a probability density function (PDF).

Probability rule P(a ≤ X ≤ b) = ∫ f(x) dx over [a, b]. P(X = k) = 0 for any exact value.
Example: adult height in cm (Normal μ=170, σ=8)

The shaded area between 162 and 178 cm represents P(162 ≤ X ≤ 178) ≈ 68%. No individual height value carries probability — only intervals do.

Distribution shape impact

Moving μ shifts the bell curve left or right without changing its shape. Increasing σ flattens and widens the curve — more probability mass spreads into the tails. Decreasing σ concentrates mass tightly around the mean, making extreme values very unlikely.

  • Mean = μ, Variance = σ² (Normal)
  • Area under the curve always equals 1
  • Real uses: heights, temperatures, measurement errors

Mixed

Point mass + continuous density

A mixed variable combines both: a real probability spike at one or more exact values, plus a continuous density elsewhere. This is not an approximation — both components are real.

Probability rule P(X = 0) = π (mass spike) and P(a ≤ X ≤ b) = (1−π) · ∫ f(x) dx for the continuous part.
Example: daily ad spend — many days £0, rest follow a skewed curve

The tall bar at £0 is a real probability mass (π = 0.45). On days when spending occurs it follows a continuous exponential-like density — two different probability mechanisms in one variable.

Distribution shape impact

Increasing π makes the zero-spike taller while flattening the continuous tail — more of the total probability sits at exactly zero. Decreasing π does the opposite: the spike shrinks and the continuous density dominates. The positive-part mean shifts where the tail mass is centered.

  • E[X] = (1−π) · μ+
  • Chart shows a bar at 0 plus a density curve
  • Real uses: insurance claims, usage data with many zeros

2. Sample space and events

Start with a fair die

A fair six-sided die has six equally likely outcomes. An event is any subset of those outcomes. If the event contains 3 faces out of 6, its probability is 3/6 = 0.5.

3. Distributions from repeated trials

Binomial distribution

If each trial has success probability p and you repeat the trial n times, the number of successes follows a binomial distribution. The chart shows how the probability mass spreads across all possible counts.

4. Prediction vs inference

Machine learning predicts. Statistical models explain.

Machine learning builds models that learn patterns from data to predict future outcomes as accurately as possible — the internal mechanism matters less than whether the prediction is correct.

Statistical models are built to interpret data and derive inferences — to estimate the size of an effect, test whether it is real, and quantify the uncertainty around that conclusion. Explanation and uncertainty communication are the primary goals, not raw predictive accuracy.

Both disciplines use probability, but they ask fundamentally different questions of the same dataset.

5. Long-run behavior

Law of large numbers

Individual outcomes are noisy. Running proportions are not. As the number of Bernoulli trials increases, the observed frequency tends to move toward the true underlying probability.

6. Functions & non-functions

The vertical line test

A function maps every input to exactly one output. No matter where you draw a vertical line on the graph, it crosses the curve at most once. A relation that is not a function fails this test — for some x, there are two or more corresponding y values.

Function

Each input maps to exactly one output

For every x in the domain there is precisely one y. A vertical line drawn anywhere on the graph crosses the curve at most once.

Vertical line test ∀x: at most one y exists such that (x, y) is on the curve
Why it matters for data science

Every model — neural network, regression formula, activation function — must map each input to a single output. Relations that are not functions must be restricted to a branch (e.g. √x returns only positive values) before they can be used in a model.

  • y = x² — parabola, one y for each x
  • y = sin(x) — oscillating but one value per x
  • y = √x — restricted to positive branch
  • y = 2x + 1 — every linear equation is a function

Not a function (Relation)

One input can map to two or more outputs

For some x values there is more than one valid y. A vertical line at such an x will cross the graph twice or more — the relation fails the vertical line test.

Vertical line test ∃x: two or more y values satisfy the relation at that x
Why it matters for data science

Probability distributions are defined using functions (PDF, CDF, PMF). If the underlying relation is not a function, the probability model breaks down. Recognising non-functions shows when a domain restriction or parametric form is needed.

  • x² + y² = 9 — circle, two y values for most x
  • x = y² — sideways parabola, upper and lower branches
  • Both need restriction before use in a model
